学习使用Hexo

使用Hexo制作个人博客网站

这里的操作步骤都是再Windows操作系统下进行。如果，你觉的很low那你可以选择再Linux操作系统下进行。

1、安装Git

为什么要使用Git呢？

因为，在后面我们要将网站上传到GitHub上。GitHub为我们提供了GitHub Page这项功能，主要用于托管个人的静态网站。如果，你懂php可以使用WordPress，懂Java的可以用Jpress等等。如果你想简单和简约，那么我强烈推荐你使用Github Page。在学习以下内容之前，请先准备好GitHub账号，如果没有请自行注册。

具体Git安装方法在这里就不再赘述了。Git官网下载地址

2、安装NodeJs

这里为什么需要Node，是因为Hexo是基于NodeJs编写的一种自动生成静态页面的Blog框架。

NodeJs官网下载地址 版本一般下载最新版就可以，根据自己电脑的系统型号来选择对应的安装包。

记得最后检查一下版本，看是否Node安装成功！

3、安装Hexo

上面的都安装好以后，接下来进行安装Hexo。

使用命令来安装Hexo，这里使用 -g 全局安装是为了可以在任何地方都能使用Hexo的一些命令。当然，你也可以不选择全局，只要去掉 -g 即可。

这样Hexo的安装就完成了。接下来就试一试吧。（对了，记得安装完要检查一下是否安装成功。使用hexo -v或者全拼hexo -version）

使用上面命令来进行初始化，init 后面的参数就是你想起的文件名。Hexo初始化的时候就会自动创建这个名称的文件夹。

安装完成后，在这个目录下就会有以下几个文件夹和多个文件

  • node_modules: 依赖包
  • public：存放生成的页面
  • scaffolds：生成文章的一些模板
  • source：用来存放你的文章
  • themes：主题（用来存放你找到的一些主题模板）
  • _config.yml: 博客的配置文件

接下来就可以启动看一下页面啦！

4、Hexo配置主题

​ 如果你对这个默认的页面感觉不太喜欢，那么在Hexo官网主题菜单中可能会有你喜欢的主题哦。

下面开始搭建主题模板的Hexo吧！

​ 首先，我们需要一个主题。在这里推荐一个用的人比较多的主题（Next）。点击链接GitHub资源地址

好了，现在开始把你从 GitHub 上 Pull 下的主题放到我们上边已经创建的文件夹的中的themes文件夹下。

​ 目前为止主题还不能使用，还需要配置一下配置文件。

​ 这里需要修改的配置文件，是Hexo的配置文件。而在，主题模板下也有一个配置文件，这两个注意一下是不同的。

​ 在Hexo的配置文件最下面找到这段theme参数修改为：theme文件夹下的主题文件名称。

配置文件中其他参数请参考Hexo官网Api。设置主题的话到主题文件夹下修改Config文件。
